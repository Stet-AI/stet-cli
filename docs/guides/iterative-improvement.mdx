Use native optimize when you have a declared search space, one lever to vary, and
a stop rule. It is a measured loop, not permission to broaden a prompt, skill,
or task corpus until a result looks better.

## Say this

<Prompt description="Plan a bounded native optimize loop" actions={["copy", "cursor"]}>
Use the Stet skill to improve this one declared harness lever with native
optimize. Propose the search space, baseline, retained slice, stop rules, and
spend boundary. Confirm the plan with me before model-token spend.
</Prompt>

## What happens

Your agent should run a bounded sequence: candidate, evaluate, check in the
result, choose one next mutation, then repeat only while the declared rules
allow it. Duration and cost depend on the search-space size, retained slice,
provider, retries, verifier, and stop rule. Before launch, require an expected
time and spend band; there is no universal band. Each change needs a one-lever
hypothesis and a record of rejected levers. Stop when evidence is invalid or
inspect-only, improvement stops, the search space is exhausted, or the next move
crosses the approved spend boundary. Do not widen the lever, task slice, or
decision claim without asking.

A finalist still needs matching holdout approval before promotion or any public
claim. Screening evidence is useful for iteration; it does not silently become
holdout evidence.

## What your agent should report back

The report should state the current candidate, one changed lever, hypothesis,
result, evidence posture, stop-rule status, observed spend, and one next action.
Its values must come from the actual Trial Result:

> **Illustrative receipt shape:** Routing: iterate, hold, or inspect. Confidence
> and posture: actual values. Coverage: evaluated denominator and retained
> evidence. Caveat: stop-rule or holdout constraint. Observed spend: actual
> value. Next action: one supported action.

Ask: “Which lever changed this round?” “What is the retained denominator?”
“Does the finalist still require holdout evidence?”

It should say explicitly when a holdout is required rather than implying that a
screening result authorizes promotion.
